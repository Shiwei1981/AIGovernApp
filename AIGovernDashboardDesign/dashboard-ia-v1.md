# AI Govern Dashboard - 信息架构与页面骨架 V1

本文档在 `dashboard-columns-v1.md` 的基础上，定义站点级信息架构、导航方式、首页骨架、二级页面骨架，以及本期实现优先级下的呈现策略。

## 1. 当前设计方向

本网站应被定义为一个 **AI Control Tower / Governance Dashboard**，而不是文档门户或报表仓库。

核心原则：

1. **决策优先**：页面首先回答“哪里有风险、哪里要行动”，而不是罗列制度。
2. **低重叠分域**：一级栏目按管理对象与治理闭环拆分，避免同一问题在多个栏目重复解释。
3. **首页总览，领域下钻**：一级页面只展示每个领域最关键的 1-2 个指标；点击某个领域后进入只对应该领域的二级页面，详细指标与图表在二级页面承接。
4. **真实数据优先，假数据补位**：本期能接微软平台真实数据的栏目优先做真数据；事件与治理类先允许假数据占位。
5. **默认周视图**：趋势图默认按周展示，支持最近 4 周 / 12 周切换。

## 2. 站点地图

```text
    ├── 5. 输出可信与内容溯源（二级页面）
    ├── 6. 验证、审计与合规保证（二级页面）
    ├── 7. 运行事件、响应与整改（二级页面）
    └── 8. 治理监督与控制执行（二级页面）
```

当前页面分层定义如下：

- **一级页面**：首页总览
- **二级页面**：每个领域各自的领域页面

暂不建议在二级页面之外再增加大量三级独立页面。  
更合适的方式是：**二级页面内，用页面内分区承载二级目录。**

## 3. 全站导航建议

## 4. 首页总览骨架

首页已调整为更轻量的 **control tower overview**，仅保留：
- 顶部状态 / 筛选条
- 8 个一级栏目卡片

不再保留：
- 全局摘要区
- 重点行动区

### 4.1 首页布局

1. **顶部状态条**
   - 时间范围
   - 最近更新时间

2. **一级栏目卡片区（8 张卡）**
    - 每张卡只展示：
      - 领域名称
      - 1-2 个首页指标
      - 必要的短说明
      - 一个状态色（绿 / 黄 / 红）
    - 单击进入该领域 dashboard
    - 大屏宽度下优先保持 **4 张卡 / 行**

### 4.2 一级页面与二级页面的指标关系

1. 一级页面中，每个领域只显示 **2 个关键指标**
2. 点击后进入的二级页面，只对应 **1 个领域**
3. 二级页面应承接该领域 **全部相关指标**
4. 一级页面的 2 个指标必须是该二级页面全部指标的 **子集**
5. 二级页面可以用**不同于一级页面**的图形或组织形式承接这些指标
6. 二级页面不要求把一级页面的 2 个指标机械重复为顶部独立卡片

### 4.3 首页卡片建议指标

| 一级栏目 | 首页展示的 1-2 个指标 | 状态 |
|---|---|---|

| 7. 运行事件、响应与整改 | 待逐项评审 | 待确认 |
| 8. 治理监督与控制执行 | 待逐项评审 | 待确认 |

## 5. 二级页面统一骨架

所有二级页面建议使用同一版式，降低学习成本。

### 5.1 页面结构

1. **顶部状态条**
   - 时间范围
   - 最近更新时间

2. **领域页头卡片**
   - 栏目名称
   - 页面定位说明
   - 风险 / 状态

3. **二级目录分区**
   - 每个二级目录一个内容区块
   - 每个区块原则上 1-4 个图表
   - 每个区块保留一句“管理意义说明”
   - 应覆盖该领域的全部相关指标，而不只覆盖首页的 2 个指标
   - 一级页面的 2 个指标必须在这些区块中被覆盖，但不要求单独重复展示
   - 每个指标 / 图形卡片应带自己的数据来源说明
   - 不再只在整个二级目录区块底部放一条共享数据来源
   - 当指标需要治理核查明细时，卡片内可包含**默认折叠**的 resource list / comparison table
   - 区块标题不再显示 `Current Phase` / `Phase 2` 之类的 phase 标签

4. **底部行动区**
   - 待处理项
   - 关联事件 / 关联整改
   - 页面级口径 / 新鲜度说明

### 5.2 图表数量控制

| 页面层级 | 建议图表数量 |
|---|---|
| 首页 | 不超过 10-12 个可见图形对象 |
| 二级页面 | 通常控制在 6-10 个图表/卡片 |
| 单个二级目录区块 | 1-4 个图表 |

## 6. 本期实现下的页面呈现策略

基于当前“是否本期实现”的确认结果，本期建议分三层呈现：

| 层级 | 呈现方式 | 说明 |
|---|---|---|
| 真数据核心区 | 首页 + 二级页面主图表 | 优先使用微软平台真实数据 |
| 假数据演示区 | 先出可视化样例 | 用于事件、治理等暂时缺乏真实接入的数据 |
| 后置区 | 保留区块但弱化展示，或在原型中直接隐藏 | 避免在区块标题上增加额外 phase 标签干扰阅读 |

对于当前阶段不准备展示、但仍需保留设计位置的二级目录区块，原型中可直接**隐藏**，而不强制显示为弱化卡片。

### 6.1 本期真数据重点栏目

优先做真数据的一级栏目：

1. AI 资产台账  
2. 第三方与供应链  
3. 数据与隐私  
4. AI 安全防护  
5. 输出可信与内容溯源  
6. 验证、审计与合规保证

### 6.2 本期假数据占位栏目

1. 运行事件、响应与整改  
2. 治理监督与控制执行
